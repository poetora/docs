每个页面都是一个 MDX 文件（后缀名为 `.mdx`），它将 Markdown 内容与 React 组件相结合，让你创建丰富、交互式的文档。

## 页面元数据

每个页面都以前言开始，即文件开头被 `---` 包围的 YAML 元数据。此元数据定义了你的页面如何显示和行为。

使用元数据来控制：

- 页面标题和描述
- 侧边栏标题、图标和标签

<ResponseField
  name="title"
  type="string"
  required
>
  页面标题，显示在导航和浏览器标签中。
</ResponseField>

<ResponseField name="description" type="string">
  此页面内容的简要描述。显示在标题下方并改善 SEO。
</ResponseField>

<ResponseField name="sidebarTitle" type="string">
  在侧边栏导航中显示的短标题。
</ResponseField>

<ResponseField name="icon" type="string">
  显示在侧边栏中页面标题旁边的图标。支持
  FontAwesome 和 Lucide 图标。
</ResponseField>

<ResponseField name="tag" type="string">
  在侧边栏中显示在页面标题旁边的标签。
</ResponseField>

```yaml 前言示例 wrap
---
title: '关于前言'
description: '前言是控制页面显示和行为的元数据'
sidebarTitle: '前言'
icon: 'book'
tag: 'NEW'
---
```

## 页面模式

使用 `mode` 设置控制页面的显示方式。

### 默认

如果未定义模式，默认为带有侧边栏导航和目录的标准布局。

```yaml
---
title: '默认页面标题'
---
```

### 宽版

宽版模式隐藏目录。这对于没有任何标题的页面或者你希望使用额外的水平空间时很有用。宽版模式适用于所有主题。

```yaml
---
title: '宽版页面标题'
mode: 'wide'
---
```

### 自定义

自定义模式提供了一个极简布局，除了顶部导航栏外移除了所有元素。自定义模式是一个空白画布，用于创建登录页或任何其他你希望具有最少导航元素的独特布局。自定义模式适用于所有主题。

```yaml
---
title: '自定义页面标题'
mode: 'custom'
---
```

### 居中

居中模式移除侧边栏和目录，使内容居中。这对于变更日志或其他你想要强调内容的页面很有用。

```yaml
---
title: '居中页面标题'
mode: 'center'
---
```

## 外部链接

使用 `url` 元数据直接从导航链接到外部站点。

```yaml
---
title: 'npm 包'
url: 'https://www.npmjs.com/package/poetora'
---
```

## 搜索引擎优化

大多数 SEO 元标签是自动生成的。你可以手动设置 SEO 元标签来改善你的站点 SEO、社交分享和浏览器兼容性。

<Note>带有冒号的元标签必须用引号包围。</Note>

```yaml
---
'twitter:image': '/images/social-preview.jpg'
---
```

查看 [SEO](/settings/seo) 了解完整的 SEO 元数据选项。

## 内部搜索关键词

通过在元数据中提供 `keywords` 来增强特定页面在内置搜索中的可发现性。这些关键词不会作为页面内容的一部分或在搜索结果中显示，但搜索它们的用户将看到该页面作为结果。

```yaml
---
keywords: ['配置', '设置', '入门']
---
```
