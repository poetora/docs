## 基础提示块

### 信息提示 (Info)

用于提供一般性信息或补充说明：

<Info>
  这是一个信息提示块，用于显示对读者有帮助的一般性信息。
</Info>

```mdx
<Info>
  这是一个信息提示块，用于显示对读者有帮助的一般性信息。
</Info>
```

### 警告提示 (Warning)

用于提醒用户注意潜在问题或重要注意事项：

<Warning>
  这是一个警告提示块，提醒用户注意可能的风险或重要事项。
</Warning>

```mdx
<Warning>
  这是一个警告提示块，提醒用户注意可能的风险或重要事项。
</Warning>
```

### 注意事项 (Note)

用于强调需要特别关注的信息：

<Note>
  这是一个注意事项提示块，用于强调读者需要特别留意的内容。
</Note>

```mdx
<Note>
  这是一个注意事项提示块，用于强调读者需要特别留意的内容。
</Note>
```

### 小贴士 (Tip)

用于分享有用的建议或小技巧：

<Tip>
  这是一个小贴士提示块，分享有助于提升效率或解决问题的建议。
</Tip>

```mdx
<Tip>
  这是一个小贴士提示块，分享有助于提升效率或解决问题的建议。
</Tip>
```

用于确认某个操作已成功完成：

### 故障排除部分

帮助用户解决常见问题：

<Warning>
**常见错误：文档无法保存**

如果遇到文档保存失败的问题，通常是由于以下原因之一：

- 网络连接不稳定
- 浏览器缓存问题
- 文档内容过大（超过 10MB）

</Warning>

<Tip>
**解决方案**

1. 检查网络连接是否正常
2. 清除浏览器缓存并刷新页面
3. 将大文档拆分为多个较小的文档
4. 如果问题持续存在，请联系客服

</Tip>

<Info>
**替代方案**

临时解决方案：可以将文档内容复制到本地文本编辑器中备份，然后重新创建文档。

</Info>

## 组件属性

### 通用 Callout 属性

<ParamField path="icon" type="string">
  自定义图标名称。支持 Font Awesome 和 Lucide 图标库中的图标，如 `"info"`、`"warning"`、`"check"` 等。
</ParamField>

<ParamField path="color" type="string">
  自定义颜色。使用十六进制色码定义提示块的主题颜色，如 `"#FF5733"`、`"#3B82F6"` 等。
</ParamField>

<ParamField path="title" type="string">
  可选的标题文本。显示在提示块顶部的自定义标题，用于突出重点信息。
</ParamField>

### 预设样式的属性

所有预设提示块（Info、Warning、Note 等）都支持以下属性：

<ParamField path="title" type="string">
  可选的自定义标题。覆盖预设样式的默认标题，显示自定义的标题文本。
</ParamField>

## 样式和主题

### 深色模式适配

所有提示块都完美支持深色模式，自动调整颜色和对比度：

<Info>
  在深色模式下，提示块会自动调整配色以确保最佳的可读性。
</Info>

<Warning>
  即使在深色主题下，警告信息仍然保持足够的视觉冲击力。
</Warning>

### 响应式设计

提示块在不同屏幕尺寸下都有良好的显示效果：

- **桌面端** - 完整显示所有内容和图标
- **平板端** - 自适应宽度，保持清晰度
- **移动端** - 紧凑布局，核心信息突出

## 最佳实践

### 选择合适的类型

- **Info** - 中性信息，不涉及成功/失败
- **Tip** - 有用的建议和技巧
- **Note** - 需要特别注意的内容
- **Warning** - 潜在风险或重要提醒
- **Check** - 成功状态或已完成的任务
- **Danger** - 严重警告或不可逆操作

### 内容编写

- **简洁明了** - 用简单的语言表达要点
- **关键信息优先** - 将最重要的信息放在前面
- **行动指导** - 告诉用户应该做什么
- **避免过度使用** - 不要让页面充满提示块

### 视觉层次

- **一页一重点** - 避免同时使用多个高优先级提示块
- **颜色平衡** - 注意不同颜色提示块的搭配
- **间距合理** - 在提示块之间留出适当间距
- **与内容协调** - 确保提示块不会打断阅读流程

## 可访问性

提示块组件支持完整的可访问性功能：

- **语义化标签** - 使用适当的 ARIA 标签
- **屏幕阅读器** - 提供清晰的内容读取
- **颜色无关性** - 不仅依赖颜色传达信息
- **键盘访问** - 支持键盘导航和操作
