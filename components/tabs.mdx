# 标签页

标签页组件用于在单个容器中组织多个内容面板，用户可以通过点击标签在不同内容之间切换。这是展示相关但独立信息的理想选择。

## 实际应用场景

### 安装和配置指南

展示不同操作系统的安装方法：

<Tabs>
  <Tab title="Windows" icon="windows">
    🪟 **Windows 系统配置**

    在 Windows 上使用 Poetora CLI 工具：

    **系统要求：**
    - Windows 10 或更高版本
    - Node.js 18.0 或更高版本
    - PowerShell 5.1 或更高版本

    **安装步骤：**

    1. **安装 Node.js**
       ```powershell
       # 下载并安装 Node.js LTS 版本
       winget install OpenJS.NodeJS
       ```

    2. **安装 Poetora CLI**
       ```powershell
       npm install -g @poetora/cli
       ```

    3. **验证安装**
       ```powershell
       poetora --version
       ```

    4. **登录账号**
       ```powershell
       poetora login
       ```

    <Tip>
    建议使用 Windows Terminal 以获得更好的命令行体验。
    </Tip>

  </Tab>
  <Tab title="macOS" icon="apple">
    🍎 **macOS 系统配置**

    在 macOS 上使用 Poetora CLI 工具：

    **系统要求：**
    - macOS 10.15 (Catalina) 或更高版本
    - Node.js 18.0 或更高版本
    - Terminal 或 iTerm2

    **安装步骤：**

    1. **安装 Homebrew（如果未安装）**
       ```bash
       /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
       ```

    2. **安装 Node.js**
       ```bash
       brew install node
       ```

    3. **安装 Poetora CLI**
       ```bash
       npm install -g @poetora/cli
       ```

    4. **验证安装**
       ```bash
       poetora --version
       ```

    5. **登录账号**
       ```bash
       poetora login
       ```

    <Info>
    macOS 用户可能需要在安装时使用 `sudo` 权限。
    </Info>

  </Tab>
  <Tab title="Linux" icon="linux">
    🐧 **Linux 系统配置**

    在 Linux 发行版上使用 Poetora CLI 工具：

    **支持的发行版：**
    - Ubuntu 20.04+
    - Debian 11+
    - CentOS 8+
    - Fedora 35+

    **Ubuntu/Debian 安装：**

    1. **更新包管理器**
       ```bash
       sudo apt update
       ```

    2. **安装 Node.js**
       ```bash
       curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
       sudo apt-get install -y nodejs
       ```

    3. **安装 Poetora CLI**
       ```bash
       sudo npm install -g @poetora/cli
       ```

    **CentOS/RHEL 安装：**

    1. **安装 Node.js**
       ```bash
       curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
       sudo yum install -y nodejs npm
       ```

    2. **安装 Poetora CLI**
       ```bash
       sudo npm install -g @poetora/cli
       ```

    **验证和登录：**
    ```bash
    poetora --version
    poetora login
    ```

    <Warning>
    某些 Linux 发行版可能需要额外的开发工具包。如遇问题请安装 `build-essential`。
    </Warning>

## 组件属性

### Tabs 容器属性

<ParamField path="defaultTab" type="number" default="0">
  默认激活的标签页索引。从 0 开始计数，用于指定页面加载时默认显示哪个标签页。
</ParamField>

<ParamField path="orientation" type="'horizontal' | 'vertical'" default="'horizontal'">
  标签页方向。控制标签页的排列方式：
  - `horizontal` - 水平排列（默认）
  - `vertical` - 垂直排列
</ParamField>

### Tab 单个标签属性

<ParamField path="title" type="string" required>
  标签页标题。显示在标签按钮上的文本，用于标识该标签页的内容。
</ParamField>

<ParamField path="icon" type="string">
  标签页图标。显示在标题前的图标名称，支持 Font Awesome 和 Lucide 图标库。
</ParamField>

<ParamField path="disabled" type="boolean" default="false">
  是否禁用该标签页。设置为 `true` 时，该标签页不可点击且显示为灰色状态。
</ParamField>


## 最佳实践

### 标签组织

- **逻辑分组** - 将相关内容组织在一起
- **标题简洁** - 使用简短而描述性的标题
- **数量适中** - 避免标签页过多影响用户体验
- **顺序合理** - 按重要性或使用频率排序

### 内容设计

- **独立完整** - 每个标签页的内容应该相对独立
- **长度均衡** - 避免某个标签页内容过长或过短
- **视觉一致** - 保持各标签页的视觉风格一致
- **交互友好** - 考虑移动设备的使用体验

### 性能优化

- **懒加载** - 对大量内容实施懒加载
- **图片优化** - 压缩标签页中的图片
- **代码分割** - 将复杂内容分割到不同标签页
- **缓存策略** - 合理缓存标签页内容

## 可访问性

标签页组件支持完整的可访问性功能：

- **键盘导航** - 支持 Tab、Arrow 键导航
- **屏幕阅读器** - 提供适当的 ARIA 标签
- **焦点管理** - 清晰的焦点指示
- **状态提示** - 明确指示当前激活的标签页
