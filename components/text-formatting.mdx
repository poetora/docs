## 标题设置

标题能够有效组织您的内容结构，创建导航锚点，帮助读者快速浏览您的文档内容。标题会自动出现在目录中，提升用户阅读体验。

### 创建标题

使用 `#` 符号创建不同级别的标题：

```mdx
## 主要章节标题

### 子章节标题

#### 小节标题
```

<Tip>
  建议使用描述性强、包含关键词的标题，清楚地说明后续内容。这样既能改善用户导航体验，也有利于搜索引擎优化。
</Tip>

### 禁用锚点链接

默认情况下，标题包含可点击的锚点链接，用户可以直接链接到特定章节。您可以通过在 HTML 或 React 标题中使用 `noAnchor` 属性来禁用这些锚点链接。

<CodeGroup>

```mdx HTML 标题示例
<h2 noAnchor>无锚点链接的标题</h2>
```

```mdx React 标题示例
<Heading level={2} noAnchor>
  无锚点链接的标题
</Heading>
```

</CodeGroup>

当使用 `noAnchor` 时，标题将不会显示锚点图标，点击标题文本也不会复制锚点链接到剪贴板。

## 文本格式化

我们支持大部分 Markdown 格式来强调和美化文本内容。

### 基础格式

为您的文本应用这些格式样式：

| 样式       | 语法       | 示例           | 效果           |
| ---------- | ---------- | -------------- | -------------- |
| **粗体**   | `**文本**` | `**重要提示**` | **重要提示**   |
| _斜体_     | `_文本_`   | `_强调内容_`   | _强调内容_     |
| ~~删除线~~ | `~文本~`   | `~已废弃功能~` | ~~已废弃功能~~ |

### 组合格式

您可以组合使用多种格式样式：

```mdx
**_粗体加斜体_**
**~~粗体加删除线~~**
_~~斜体加删除线~~_
```

**_粗体加斜体_**<br />
**~~粗体加删除线~~**<br />
_~~斜体加删除线~~_

### 上标和下标

对于数学表达式或脚注，使用 HTML 标签：

| 类型 | 语法              | 示例               | 效果             |
| ---- | ----------------- | ------------------ | ---------------- |
| 上标 | `<sup>文本</sup>` | `示例<sup>2</sup>` | 示例<sup>2</sup> |
| 下标 | `<sub>文本</sub>` | `示例<sub>n</sub>` | 示例<sub>n</sub> |

## 链接设置

链接帮助用户在页面间导航和访问外部资源。使用描述性的链接文本能够提高可访问性和用户体验。

### 内部链接

使用根相对路径链接到文档中的其他页面：

```mdx
[快速开始](/quickstart)
[步骤组件](/components/steps)
```

[快速开始](/quickstart)<br />
[步骤组件](/components/steps)

<Note>
  避免使用相对链接如 `[页面](../page)`，因为它们加载较慢且无法得到有效优化。推荐使用根相对路径。
</Note>

### 外部链接

对于外部资源，包含完整的 URL：

```mdx
[Markdown 指南](https://www.markdownguide.org/)
```

[Markdown 指南](https://www.markdownguide.org/)

## 引用块

引用块能够突出显示重要信息、引用内容或示例。

### 单行引用

在文本前添加 `>` 创建引用块：

```mdx
> 这是一段从主要内容中突出显示的引用。
```

> 这是一段从主要内容中突出显示的引用。

### 多行引用

对于较长的引用或多个段落：

```mdx
> 这是多行引用的第一段。
>
> 这是第二段，通过包含 `>` 的空行分隔。
```

> 这是多行引用的第一段。
>
> 这是第二段，通过包含 `>` 的空行分隔。

<Tip>
  谨慎使用引用块以保持其视觉冲击力和意义。对于注释、警告和其他信息，考虑使用
  [提示组件](/components/callouts)。
</Tip>

## 数学表达式

我们支持 LaTeX 语法来渲染数学表达式和公式。

### 行内数学

使用单个美元符号 `$` 来编写行内数学表达式：

```mdx
勾股定理表明在直角三角形中有 $(a^2 + b^2 = c^2)$ 的关系。
```

勾股定理表明在直角三角形中有 $(a^2 + b^2 = c^2)$ 的关系。

### 块级公式

使用双美元符号 `$$` 来创建独立的公式：

```mdx
$$
E = mc^2
$$
```

$$
E = mc^2
$$

<Info>
  LaTeX 支持需要正确的数学语法。请参考 [LaTeX
  文档](https://www.latex-project.org/help/documentation/) 获取完整的语法指南。
</Info>

## 换行和间距

控制间距和换行以提高内容的可读性。

### 段落分隔

使用空行分隔段落：

```mdx
这是第一段。

这是第二段，通过空行分隔。
```

这是第一段。

这是第二段，通过空行分隔。

### 手动换行

在段落内使用 HTML `<br />` 标签强制换行：

```mdx
这行在这里结束。<br />
这行从新行开始。
```

这行在这里结束。<br />
这行从新行开始。

<Tip>在大多数情况下，使用空行进行段落分隔比手动换行提供更好的可读性。</Tip>

## 最佳实践

### 内容组织

- 使用标题创建清晰的内容层次结构
- 遵循正确的标题层级（不要从 H2 跳到 H4）
- 编写描述性、包含关键词的标题文本

### 文本格式化

- 使用粗体来强调，而不是整个段落
- 为术语、标题或微妙强调保留斜体
- 避免过度格式化，以免分散内容注意力

### 链接设置

- 编写描述性链接文本，而不是"点击这里"或"阅读更多"
- 对内部链接使用根相对路径
- 定期测试链接以防止断开引用
