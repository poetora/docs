## 基础用法

### 简单步骤

创建基本的步骤指南：

<Steps>
  <Step title="注册账号">
    访问 Poetora 官网，点击"立即注册"按钮开始创建你的账号。

    需要提供：
    - 有效的邮箱地址
    - 安全的密码（至少8位字符）
    - 用户名或真实姓名

  </Step>
  <Step title="验证邮箱">
    检查你的邮箱，点击验证邮件中的确认链接完成账号激活。

    如果没有收到邮件，请：
    - 检查垃圾邮件文件夹
    - 确认邮箱地址输入正确
    - 等待几分钟后重新发送

  </Step>
  <Step title="创建工作空间">
    登录后创建你的第一个工作空间，设置项目名称和基本信息。

    工作空间设置包括：
    - 工作空间名称
    - 团队成员邀请
    - 权限配置
    - 主题选择

  </Step>
</Steps>

### 带图标的步骤

为步骤添加图标增强视觉识别：

<Steps>
  <Step title="安装依赖" icon="download">
    首先安装项目所需的依赖包：

    ```bash
    npm install @poetora/sdk
    # 或使用 yarn
    yarn add @poetora/sdk
    ```

    这将安装 Poetora SDK 和所有必要的依赖项。

  </Step>
  <Step title="配置 API" icon="key">
    在项目中配置 API 密钥：

    ```javascript
    import { PoetoraClient } from '@poetora/sdk';

    const client = new PoetoraClient({
      apiKey: process.env.POETORA_API_KEY,
      baseURL: 'https://api.poetora.com/v1'
    });
    ```

    <Warning>
    请确保将 API 密钥存储在环境变量中，不要直接写在代码里。
    </Warning>

  </Step>
  <Step title="初始化编辑器" icon="file-pen-line">
    在你的应用中初始化 Poetora 编辑器：

    ```javascript
    const editor = await client.createEditor({
      containerId: 'poetora-editor',
      document: {
        id: 'your-document-id',
        title: '我的文档'
      }
    });

    editor.on('save', (content) => {
      console.log('文档已保存:', content);
    });
    ```

  </Step>
  <Step title="测试功能" icon="check">
    测试编辑器的基本功能：

    1. 创建新文档
    2. 添加一些内容
    3. 保存文档
    4. 验证数据同步

    如果一切正常，你就可以开始使用 Poetora 了！

  </Step>
</Steps>

## 实际应用场景

### 产品使用教程

详细的产品功能介绍：

<Steps>
  <Step title="创建第一个文档" icon="file-plus">
    🎯 **目标：** 创建并编辑你的第一个文档

    **操作步骤：**
    1. 点击工作台左上角的"新建文档"按钮
    2. 从模板库中选择合适的模板，或选择"空白文档"
    3. 输入文档标题，例如"产品需求文档"
    4. 选择文档所属的文件夹和标签

    **提示：** 选择合适的模板可以帮你快速开始，节省格式设置时间。

    <Info>
    新用户建议从"快速开始"模板开始，这个模板包含了常用的组件示例。
    </Info>

  </Step>
  <Step title="掌握基础编辑" icon="square-pen">
    ✍️ **目标：** 学会使用编辑器的基本功能

    **核心功能：**
    - **文本格式化** - 粗体、斜体、下划线等
    - **标题层级** - H1 到 H6 的标题设置
    - **列表创建** - 有序和无序列表
    - **链接插入** - 内部链接和外部链接

    **快捷键练习：**
    ```
    Ctrl/Cmd + B    粗体
    Ctrl/Cmd + I    斜体
    Ctrl/Cmd + K    插入链接
    /               打开组件菜单
    ```

    **练习任务：**
    1. 创建一个三级标题
    2. 添加一段粗体文本
    3. 插入一个外部链接
    4. 创建一个简单的列表

  </Step>
  <Step title="使用高级组件" icon="puzzle">
    🧩 **目标：** 学会使用 Poetora 的特色组件

    **必学组件：**

    **提示框组件：**
    - 用于重要信息提醒
    - 输入 `/info` 快速插入
    - 支持多种类型：信息、警告、成功等

    **卡片组件：**
    - 用于内容分组展示
    - 支持图标和链接
    - 适合做功能介绍

    **代码块：**
    - 支持语法高亮
    - 多种编程语言
    - 可添加文件名标题

    **实践练习：**
    1. 添加一个信息提示框
    2. 创建两个并排的卡片
    3. 插入一段代码示例

  </Step>
  <Step title="团队协作功能" icon="users">
    👥 **目标：** 体验多人协作编辑功能

    **邀请团队成员：**
    1. 点击右上角的"分享"按钮
    2. 选择"邀请协作者"
    3. 输入同事的邮箱地址
    4. 设置相应的权限级别

    **权限级别说明：**
    | 权限 | 查看 | 编辑 | 评论 | 管理 |
    |------|------|------|------|------|
    | 查看者 | ✅ | ❌ | ✅ | ❌ |
    | 编辑者 | ✅ | ✅ | ✅ | ❌ |
    | 管理员 | ✅ | ✅ | ✅ | ✅ |

    **协作技巧：**
    - 使用评论功能进行讨论
    - 关注实时光标显示
    - 查看修改历史记录
    - 合理分配编辑权限

  </Step>
  <Step title="发布和分享" icon="share">
    📤 **目标：** 将文档发布并分享给目标读者

    **发布选项：**

    **公开发布：**
    - 生成公开访问链接
    - 支持 SEO 优化
    - 可设置自定义域名

    **私密分享：**
    - 密码保护访问
    - 限时访问链接
    - 特定用户访问

    **导出功能：**
    - PDF 格式导出
    - HTML 静态文件
    - Markdown 格式
    - Word 文档格式

    **发布步骤：**
    1. 点击右上角"发布"按钮
    2. 选择发布类型和访问权限
    3. 设置 SEO 信息（标题、描述等）
    4. 确认发布并获取分享链接

    <Check>
    恭喜！你已经掌握了 Poetora 的基本使用方法。现在可以开始创建专业的文档了。
    </Check>

  </Step>
</Steps>

## 组件属性

### Steps 容器属性

<ParamField path="orientation" type="'vertical' | 'horizontal'" default="'vertical'">
  步骤方向。控制步骤的排列方式：
  - `vertical` - 垂直排列（默认）
  - `horizontal` - 水平排列
</ParamField>

<ParamField path="current" type="number">
  当前激活的步骤索引。从 0 开始计数，用于高亮显示当前正在执行的步骤。
</ParamField>

<ParamField path="showNumbers" type="boolean" default="true">
  是否显示步骤编号。设置为 `false` 时隐藏步骤数字，仅显示图标或状态指示器。
</ParamField>

### Step 单个步骤属性

<ParamField path="title" type="string" required>
  步骤标题。显示在每个步骤中的主要文本，用于描述该步骤的内容或操作。
</ParamField>

<ParamField path="icon" type="string">
  步骤图标。显示在步骤标题旁的图标名称，支持 Font Awesome 和 Lucide 图标库。
</ParamField>

<ParamField path="status" type="'pending' | 'current' | 'completed' | 'error'" default="'pending'">
  步骤状态。控制步骤的视觉状态：
  - `pending` - 待执行状态（默认）
  - `current` - 当前正在执行
  - `completed` - 已完成状态
  - `error` - 错误状态
</ParamField>

## 最佳实践

### 内容组织

- **逻辑顺序** - 按实际操作顺序安排步骤
- **粒度适中** - 每个步骤包含合理的操作量
- **独立完整** - 每个步骤应该是可独立执行的
- **清晰标题** - 使用动词开头的行动性标题

### 视觉设计

- **图标一致** - 使用统一风格的图标系统
- **状态明确** - 清楚显示步骤的完成状态
- **进度指示** - 让用户了解整体进度
- **重点突出** - 突出显示当前步骤

### 用户体验

- **渐进式披露** - 逐步展示复杂信息
- **错误处理** - 为可能的错误提供解决方案
- **替代路径** - 为不同情况提供备选方案
- **反馈机制** - 及时反馈操作结果

### 内容编写

- **行动导向** - 使用明确的行动指令
- **简洁明了** - 避免冗长的描述
- **包含示例** - 提供具体的代码或操作示例
- **预期结果** - 说明每步完成后的预期效果

## 可访问性

步骤组件支持完整的可访问性功能：

- **语义化结构** - 使用适当的 HTML 语义标签
- **键盘导航** - 支持键盘在步骤间导航
- **屏幕阅读器** - 提供清晰的步骤描述
- **状态播报** - 明确播报步骤状态变化
