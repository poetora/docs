## 基础用法

### 两列布局

最常用的两列并排布局：

<Columns cols={2}>
  <div>
    **左侧内容**
    这是左侧列的内容。可以包含文本、图片、组件等任何内容。列布局会自动调整宽度，确保内容在不同屏幕上都有良好的显示效果。
  </div>
  <div>
    **右侧内容**
    这是右侧列的内容。两列之间会自动保持适当的间距，创造整洁的视觉效果。在移动设备上，列会自动堆叠为单列显示。
  </div>
</Columns>

```mdx
<Columns cols={2}>
  <div>**左侧内容** 这是左侧列的内容...</div>
  <div>**右侧内容** 这是右侧列的内容...</div>
</Columns>
```

## 组件属性

<ParamField path="cols" type="number" default="2">
  列数。设置分栏布局的列数，支持 1-6 列。数值越大，每列宽度越窄。
</ParamField>

<ParamField path="gap" type="'small' | 'medium' | 'large'" default="'medium'">
  列间距大小。控制列与列之间的间距：
  - `small` - 较小间距
  - `medium` - 中等间距（默认）
  - `large` - 较大间距
</ParamField>

<ParamField path="align" type="'start' | 'center' | 'end' | 'stretch'" default="'stretch'">
  列高度对齐方式。控制不同高度列的对齐方式：
  - `start` - 顶部对齐
  - `center` - 居中对齐
  - `end` - 底部对齐
  - `stretch` - 拉伸至相同高度（默认）
</ParamField>

## 高级用法

### 不等宽列

创建不同宽度的列布局：

<Columns cols={3}>
  <div style={{ gridColumn: 'span 2' }}>
    **主要内容区域**
    这是一个较宽的内容区域，占据了两列的宽度。这里可以放置主要的文章内容、详细说明或重要信息。
    通过设置 `gridColumn: 'span 2'`，这个区域会跨越两个列的宽度，给重要内容更多的展示空间。
  </div>
  <div>
    **侧边栏**

    这是侧边栏区域，通常用于放置：
    - 导航链接
    - 相关文章
    - 广告内容
    - 快速操作

  </div>
</Columns>

```mdx
<Columns cols={3}>
  <div style={{ gridColumn: 'span 2' }}>
    **主要内容区域** 这是一个较宽的内容区域...
  </div>
  <div>**侧边栏** 这是侧边栏区域...</div>
</Columns>
```

## 最佳实践

### 内容组织

- **逻辑分组** - 将相关内容放在相邻的列中
- **重要性排序** - 将最重要的内容放在最左侧
- **平衡布局** - 确保各列内容量相对均衡
- **响应式考虑** - 考虑在不同屏幕尺寸下的显示效果

### 视觉设计

- **对齐一致** - 保持各列内容的对齐方式一致
- **间距统一** - 使用一致的内边距和外边距
- **高度平衡** - 避免某一列内容过长或过短
- **色彩协调** - 保持各列的色彩风格协调

### 响应式适配

- **移动优先** - 优先考虑移动设备的显示效果
- **断点设置** - 在不同屏幕尺寸下调整列数
- **内容优先级** - 在小屏幕上优先显示重要内容
- **交互优化** - 确保在触摸设备上的良好交互

### 性能优化

- **图片优化** - 压缩和优化列中的图片资源
- **内容懒加载** - 对大量内容实施懒加载策略
- **CSS 优化** - 避免复杂的 CSS 选择器
- **布局稳定** - 避免内容加载时的布局跳动

## 可访问性

列布局组件支持完整的可访问性功能：

- **语义化结构** - 使用适当的 HTML 语义标签
- **键盘导航** - 支持键盘在列间导航
- **屏幕阅读器** - 提供清晰的内容结构描述
- **焦点管理** - 合理的焦点顺序和指示

### 屏幕阅读器支持

- 正确的标题层次结构
- 适当的地标角色（landmark roles）
- 清晰的内容分组
- 有意义的链接文本
